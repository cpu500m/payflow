# Podongdong's pay 도메인 모델

## Podongdong's pay 도메인
- Podongdong's pay는 네이버페이, 카카오페이와 같은 간편결제 플랫폼이다.
- 디지털 지갑 (본 프로젝트에서는 편의상 지갑이라고 지칭)에 돈을 넣어두고, 가맹점에서 결제하는 서비스이다.
  - 플랫폼은 가맹점으로부터 수수료를 받아 매출을 확보한다. 
- 포동동's pay를 사용하려면 플랫폼의 회원이 되어야 한다.
  - 회원이 되면 지갑이 자동으로 생성되고, 회원 한명당 하나의 지갑만을 가질 수 있다.
- 가맹점이 포동동's pay를 사용하려면 플랫폼과 계약을 맺어야 한다.
  - 가맹점 신청 후 관리자 승인을 받아야 하며, 승인 시 가맹점 상태가 활성 상태로 변경된다.
  - 계약 만료 일시는 매일 00시를 기준으로 만료처리된다.
  - 활성 상태의 가맹점만 결제를 수신할 수 있다.
    - 결제 시점에 계약 만료일이 지난 가맹점은 결제를 수신할 수 없다.
  - 가맹점에 문제가 있을 시 정지될 수 있다.
  - 계약이 끝날 시 만료상태로 변경된다.
  - 수수료율은 가맹점마다 상이하다.
  - 만료/정지된 가맹점이라도 만료/정지 이전에 승인된 결제에 대한 정산은 정상적으로 처리된다.
- 충전은 은행 계좌에서 지갑으로 돈을 옮김으로써 이루어진다.
- 지갑으로 들어오고 나가는 모든 내역은 거래 원장에 반드시 기록되어야 한다.
  - 거래 원장은 한번 기록되면 수정하거나 삭제할 수 없다.
- 회원은 지갑에 돈이 있어야 결제를 할 수 있다. 
  - 결제 요청 시, 지갑에 잔액이 충분한지 확인 후 충분하다면 결제가 승인된다.
  - 잔액이 부족하면 결제가 실패 처리된다.
  - 어떠한 경우에도 돈이 중복 출금 되어서는 안된다.
- 사용자는 승인된 결제에 대해 취소를 요청할 수 있다.
  - 정산이 완료되기전의 승인된 결제에 한해서만 취소 가능하다.
- 매일 새벽2시 배치로 정산 행위가 이루어진다.
  - 전날 승인된 결제를 가맹점별로 모아 수수료를 떼고 정산금을 지급한다.
- 결제완료, 충전완료, 정산완료 등의 이벤트가 발생 시 사용자/가맹점에게 알림을 발송한다.
  - 알림은 브라우저를 통해 이루어진다. (토스트팝업 + 알림 1건 추가)

## 회원 service
### 회원(Member)
_Entity_
#### 속성
- `id`: `Long`
- `email`: 이메일 - Natural ID
- `nickname`: 닉네임
- `encodedPassword`: 비밀번호
- `status`: `MemberStatus`
- `registeredAt` : 가입 시각
#### 행위
- `static register()` : 가입
- `changeNickname()` : 닉네임 변경
- `verifyPassword()` : 비밀번호 검증
- `changePassword()` : 비밀번호 변경
- `suspend()` : 정지
- `resume()` : 재개
- `deactivate()` : 탈퇴
#### 규칙
- 회원 가입 시 즉시 활성상태가 된다.
- 활성상태에서 탈퇴가 가능하다.
  - 지갑에 잔액이 남아있으면 탈퇴 불가능하다.
- 비밀번호는 해시를 만들어서 저장한다.
- 상태는 아래와 같이 전이 가능하다.
  - `ACTIVE` > `DEACTIVATED` , `SUSPENDED`
  - `SUSPENDED` > `ACTIVE` , `DEACTIVATED`
### 회원 상태(MemberStatus)
_Enum_
#### 상수
- `ACTIVE`: 활성
- `SUSPENDED`: 정지
- `DEACTIVATED`: 탈퇴

### 비밀번호 인코더(PasswordEncoder)
_Domain Service_
#### 행위
- `encode()` : 암호화
- `matches()` : 비밀번호가 일치하는지 확인

### Email
_Value Object_
### 속성
- `address`: 주소

## 가맹점 service

### 가맹점(Merchant)
_Entity_
#### 속성
- `id` : `Long`
- `name` : 사업자명
- `commissionRate` : 수수료율
- `status` : `MerchantStatus`
- `registeredAt` : 등록 시각
- `activatedAt` : 활성 시각
- `contractStartDate` : 계약 시작일
- `contractEndDate` :  계약 만료일
- `suspendedAt` : 정지 시각
#### 행위
- `static register()`: 가맹점 등록
- `activate()`: 승인 완료
- `suspend()`: 계약 정지
- `resume()`: 계약 재개
- `expire()`: 계약 만료
- `isActive()` : 활성화 여부 검사
#### 규칙
- 가맹점 등록 후 상태는 승인 대기 상태
- 관리자의 승인 후 활성 상태가 된다.
- 활성 상태 외에는 결제를 수신할 수 없다.
- 가맹점 및 플랫폼은 계약을 중도 해지 할 수 없다.
  - 만료일이 지나면 만료 처리 된다.
    - 계약 만료는 매일 00시 배치에 의해 자동으로 처리된다.
- 상태는 아래와 같이 전이 가능하다.
  - `PENDING` > `ACTIVE`
  - `ACTIVE` > `SUSPENDED` , `EXPIRED`
  - `SUSPENDED` > `ACTIVE` , `EXPIRED`
### 가맹점 상태(MerchantStatus)
_Enum_
#### 상수
- `PENDING`: 승인 대기
- `ACTIVE`: 활성
- `SUSPENDED`: 정지
- `EXPIRED`: 만료

## 지갑 service

### 지갑(Wallet)
_Entity_

#### 속성
- `id` : `Long`
- `balance` : 잔액
- `createdAt` : 생성 시각
- `memberId` : 회원 ID
#### 행위
- `static create()` : 지갑 생성
- `charge(long amount)` : 충전
- `deduct(long amount)` : 차감
- `refund(long amount)` : 환불
#### 규칙
- 회원 한명당 지갑 하나만을 가진다.
  - 회원 생성 시점에 지갑이 자동 생성된다.
- 지갑에 잔액이 충분해야 결제가 가능하다.

### 거래(WalletTransaction)
_Entity_
#### 속성
- `id` : `Long`
- `type` : 거래 유형
- `amount` : 거래 금액
- `description` : 설명
- `balanceAfter` : 거래 후 금액
- `createdAt` : 거래 시각
- `walletId` : 지갑 ID
#### 행위
- `static create` : 거래 내역 생성
#### 규칙
- 지갑으로 들어오고 나가는 모든 내역은 거래 원장에 반드시 기록되어야 한다.
  - 충전/차감/환불 시 거래원장이 함께 생성된다.
- 거래 원장은 한번 기록되면 수정하거나 삭제할 수 없다. 

### 거래 유형(TransactionType)
_Enum_
#### 상수
- `CHARGE`: 충전
- `DEDUCT`: 차감
- `REFUND`: 환불
